# Как создать APK для проекта AgroNexus

Ваш проект уже почти готов к сборке. Мы будем использовать **Expo EAS Build**, так как это самый простой и надежный способ создать APK файл для React Native приложения на базе Expo.

## Что уже настроено:
- В файле `app.json` указан ID проекта и уникальный пакет (`com.temursat.AgroNexus`).
- В файле `eas.json` настроен профиль `preview`, который специально сконфигурирован для сборки **APK** файла (а не AAB для Google Play).

## Шаги, которые вам нужно выполнить:

### 1. Установка EAS CLI
Если вы еще не устанавливали инструменты Expo для сборки, выполните в терминале:
```powershell
npm install -g eas-cli
```

### 2. Вход в аккаунт Expo
Вам нужно войти в свою учетную запись Expo (если ее нет, создайте на [expo.dev](https://expo.dev)):
```powershell
eas login
```

### 3. Запуск сборки
Выполните следующую команду для запуска процесса создания APK:
```powershell
eas build --platform android --profile preview
```

### 4. Что будет происходить дальше:
1. **Проверка проекта**: EAS предложит настроить проект в облаке (если спросит "Would you like to automatically configure?", ответьте **Yes**).
2. **Создание ключей (Keystore)**: EAS спросит, хотите ли вы сгенерировать новый Keystore. Если вы собираете приложение впервые, выберите **Generate new keystore**.
3. **Очередь сборки**: Ваше приложение будет отправлено на сервера Expo для компиляции. Это займет от 5 до 15 минут.
4. **Ссылка на скачивание**: По завершении сборки в терминале появится ссылка, а также QR-код. Откройте ссылку или отсканируйте код телефоном, чтобы скачать готовый **APK** файл.

---

> [!TIP]
> **Почему используется профиль `preview`?**
> Стандартная команда `eas build -p android` создает файл формата `.aab`, который нужен для Google Play. Профиль `preview` настроен на создание `.apk`, который можно просто установить на телефон для тестирования.

> [!IMPORTANT]
> Убедитесь, что у вас стабильное интернет-соединение во время выполнения команды, так как проект будет загружаться в облако Expo.
